# 第 6 章：注意力机制——Token 之间的信息交流协议

> **核心命题**：自注意力是一种基于内容的动态路由机制：Q（我在找什么）、K（我包含什么）、V（我传送什么）。这个三元组让 token 能够根据语义相关性自由通信，比任何固定的聚合方式都强得多。

---

## 思维框架

语言有一个核心挑战：一个词的意义，取决于它旁边的词。

"bank" 在 "river bank" 和 "investment bank" 里意思完全不同。"他" 指向的是哪个人，取决于前面几句话的语境。一段代码里的变量，需要知道它在哪里被定义、在哪里被修改。

语言模型需要解决的问题是：**如何让每个 token 在处理时，能够获取它需要的其他 token 的信息**？

最简单的解法：对所有过去的 token 取平均。把前 n 个 token 的表示向量加起来除以 n，作为当前 token 的上下文信息。

这有两个致命缺陷：

1. **没有位置区分**：第 1 个 token 和第 10 个 token 对结果贡献相等。
2. **没有内容区分**：无论语义相关还是不相关，所有 token 权重相同。

真正需要的是：**基于内容的、可学习的、动态的聚合权重**。这正是自注意力的核心设计。

---

## 理论基础

**从加权平均到自注意力**

Karpathy 在构建 GPT 时，用了一个非常漂亮的渐进式演示，分三步展示了注意力机制的发展：

**步骤 1：简单平均（基线）**
对所有过去 token 的表示取均值。代码约 5 行，效果差，但建立了"聚合过去信息"的基本概念。

**步骤 2：因果掩码加权平均**
引入下三角矩阵（tril），强制每个位置只能看到自己和之前的 token（不能"偷看"未来）。这是**因果掩码**，语言生成必须遵守的约束。

实现：创建一个下三角矩阵，上三角设为负无穷，经过 softmax 后上三角变为 0——这样每个位置只对过去的 token 有非零权重。

**步骤 3：可学习的权重——Query、Key、Value**

核心问题：权重不该是固定均匀的，应该是**数据驱动的**。

解法：为每个 token 学习三个投影：

- **Query（查询）**：这个 token 在寻找什么类型的信息？
- **Key（键）**：这个 token 广播它包含什么类型的信息？
- **Value（值）**：如果被选中，这个 token 实际传送什么内容？

注意力权重计算：`attention = softmax(Q · Kᵀ / sqrt(d_k)) · V`

其中 `Q · Kᵀ` 计算每对 token 之间 Query 和 Key 的匹配程度——匹配度高的 token 对，注意力权重大，Value 的传递更多。

**为什么要除以 sqrt(d_k)？**

这不是任意的技巧。当向量维度 d_k 很大时，Q 和 K 的内积在量级上会很大，导致 softmax 的输出接近 one-hot 分布（几乎所有权重集中在一个 token 上），梯度几乎为 0。除以 `sqrt(d_k)` 把内积的期望方差归一化到 1，保持 softmax 输出的多样性，梯度流动顺畅。

**多头注意力**

单个注意力头只能学习一种 token 交互模式。多头注意力并行运行多个头（比如 8 个头），每个头独立学习 Q、K、V 投影，捕捉不同类型的关系（句法关系、语义关系、远程依赖……），最后把所有头的输出拼接起来再投影。

这大幅提升了模型能同时建模的关系类型数量。

## 案例

> **案例：三种等价实现的对比演示**
>
> **情境**：Karpathy 想让学生看清楚因果掩码的本质，而不只是记住一行代码
>
> **关键时刻**：他并排写出三种实现：(1) 显式循环——对每个位置 t，对前 t 个 token 的表示取平均；(2) 矩阵版本——用归一化的下三角矩阵直接矩阵乘法；(3) softmax 版本——tril 矩阵上三角填 -inf，然后 softmax。三种方式的数值输出完全相同。
>
> **结果**：学生理解了因果掩码不是一个特殊的"魔法参数"，而只是"不让信息从未来流向过去"这个约束的数学表达。第三种形式在自注意力中最有用，因为它自然地允许非均匀的、数据依赖的权重。
>
> **启示**：同一个概念的多种等价实现，是建立深度理解的强大方式。当三种写法产生相同结果时，你真正理解了这个概念的本质。
>
> **可迁移**：学习新算法时，尝试用至少两种不同方式实现它，然后验证它们的输出相同。这个练习会加速你的理解速度。

---

> **案例：Q·K 内积可视化**
>
> **情境**：训练完一个小型 Transformer 后，Karpathy 把注意力权重可视化出来——一个热力图，行是当前 token，列是被关注的过去 token
>
> **关键时刻**：对于一个在生成英语的模型，你可以看到某些头在关注远处的名词（主语-谓语关系），某些头在关注相邻词（局部句法关系），某些头在关注句首的位置（全局上下文）
>
> **结果**：注意力图让"模型在学什么"变得可视化。这是现代 Transformer 可解释性研究的起点。
>
> **启示**：自注意力不是均匀地关注所有 token——它学会了有意义的、结构化的关注模式。
>
> **可迁移**：当你的 Transformer 模型输出不符合预期时，可视化注意力权重往往能提供有价值的调试线索。

## 在你的生活中

**建立对注意力机制的直觉：**

**1. 手写自注意力的核心公式（15 分钟）**
只用 NumPy，实现：`scores = Q @ K.T / sqrt(d_k)`，用 `-inf` 掩码上三角，`softmax`，然后 `scores @ V`。用随机向量测试，验证输出维度正确。这个练习把公式变成直觉。

**2. 理解 Q/K/V 的直觉意义**
思考一下：在你阅读这句话时，你的大脑是如何决定哪些词和当前词有关的？你是在模糊地做 Query-Key 匹配——"我现在处理的这个词，需要寻找什么类型的信息来理解它？"自注意力是这个过程的一个可计算的近似。

**3. 对比 encoder-only 和 decoder-only 的区别**
BERT 用的是无掩码的双向注意力——每个 token 可以看到所有其他 token，适合分类任务。GPT 用的是因果（单向）注意力——每个 token 只能看过去，适合生成任务。理解这个区别，就理解了为什么同样是 Transformer，BERT 和 GPT 的应用场景如此不同。
