# 第 10 章：从 GPT-2 到 ChatGPT——规模定律与现代训练实践

> **核心命题**：GPT-2 和 ChatGPT 架构几乎相同，差异只在规模和数据。规模定律告诉我们：给更多数据和算力，模型损失会可预测地下降。Karpathy 的 GPT-2 复现项目是理解这条定律最具体的路径。

---

## 思维框架

2024 年，Karpathy 发起了一个项目："Let's reproduce GPT-2"——用现代工具，从零复现 2019 年的 GPT-2（1.24 亿参数版本），并在 WebText 数据集的等价数据上验证训练结果。

这个项目的意义不只是"复现一个旧模型"。

它的价值在于：把 GPT-2 和 GPT-4 之间"不可知的黑暗"照亮了一部分。当你亲手实现了 GPT-2 的训练代码，你就理解了：从这里到 ChatGPT，主要发生了什么变化？

答案令人意外地简单：**主要是规模和数据**。

架构层面的变化极其微小——GPT-2 到 GPT-3 的改动只有几处技术细节（Pre-Norm 替换 Post-Norm，GELU 替换 ReLU，旋转位置编码等）。让性能提升了几个数量级的，是参数量从 124M 增长到 175B（GPT-3）再到估计数万亿（GPT-4），以及训练数据量的同等增长。

这就是规模定律的核心主张：**给更多算力和数据，模型性能会按照可预测的幂律关系提升**。

---

## 理论基础

**GPT-2 复现：每个训练细节**

Karpathy 的 GPT-2 复现涵盖了现代 LLM 训练的所有关键技术：

**混合精度训练（bfloat16）**：用 16 位浮点数而不是 32 位进行大部分计算，内存占用减半，速度大幅提升。bfloat16 相比 float16 的优势是数值范围更大，不容易发生梯度下溢。

**梯度裁剪（Gradient Clipping）**：每步更新前，检查梯度范数，如果超过阈值（通常 1.0）就缩放梯度，防止单步更新过大导致训练发散。Karpathy 展示了不加梯度裁剪时偶尔出现的损失"尖峰"，以及加上之后的平滑曲线对比。

**余弦学习率调度（Cosine LR Schedule）**：不用固定学习率，而是在训练开始时用线性预热（几百步从 0 升到峰值），然后按余弦曲线缓慢下降到接近 0。这个调度方式经验上比固定学习率收敛更好。

**权重衰减（Weight Decay）**：正则化技术，对权重施加 L2 惩罚，防止过拟合。但在 Transformer 中，不是所有参数都应该有权重衰减——1D 参数（偏置、LayerNorm 的 γ 和 β）通常不加。

**Flash Attention**：由 Tri Dao 开发的高效注意力实现，通过 GPU 内存访问模式的优化，在数学上等价但实际运行速度快数倍，内存占用减少。Karpathy 在复现中直接使用了 PyTorch 内置的 `scaled_dot_product_attention`，它底层调用 Flash Attention。

**分布式训练**：单块 GPU 的显存和算力不够训练大模型。PyTorch 的 DataParallel 和 DistributedDataParallel（DDP）让同一份代码可以在多块 GPU 上并行运行，每块 GPU 处理不同的数据批次，定期同步梯度。

**规模定律（Scaling Laws）**

2020 年，Kaplan 等人发现了语言模型性能与计算量、数据量、模型参数的幂律关系：

`L(N, D) ≈ A·N^(-α) + B·D^(-β) + 常数`

其中 L 是测试损失，N 是参数量，D 是训练 token 数。

更直接的版本（Chinchilla 定律，2022）：对于给定的计算预算 C，最优的分配是：
- 模型参数 N 和训练 token 数 D 应该大致等比例增长
- N ≈ D / 20（每个参数需要约 20 个训练 token）

这意味着，Llama 2 70B 理论上需要约 1.4 万亿个训练 token 才能达到"最优"（Llama 2 实际用了 2 万亿）。很多早期大模型（包括 GPT-3）是"欠训练"的——参数很多，但没有给足够的数据。

**涌现能力（Emergent Capabilities）**

规模定律预测的是平均损失，但实际上当模型超过某个规模时，某些**特定能力**会突然出现，而不是平滑地从小模型外推。

少样本学习（In-context Learning）就是这样一种涌现能力——小模型几乎不会，大模型在没有参数更新的情况下，只通过上下文中的示例就能学会新任务。这种能力在 GPT-2 规模（124M）时几乎没有，在 GPT-3（175B）时显著出现。

这种"涌现"是规模定律框架内最神秘也最令人兴奋的现象。

## 案例

> **案例：GPT-2 复现的具体里程碑**
>
> **情境**：Karpathy 公开直播了整个复现过程（约 4 小时），每一步都可以看到代码和训练曲线
>
> **关键时刻**：在 8 块 A100 GPU 上训练约 1 小时后，验证损失开始接近 GPT-2 原始论文的报告值（约 3.0 在 WebText 验证集）。更重要的是，他展示了每个训练技巧（Flash Attention、混合精度、学习率调度）带来的性能提升，以及不加这些技巧时的退化结果。
>
> **结果**：整个复现项目的代码只有约 1000 行 Python，却能产生与原始 GPT-2 相当的结果。这证明了：理解 GPT-2 = 理解现代 LLM 的核心。
>
> **启示**：你不需要 OpenAI 的资源才能理解他们做了什么。一台配置合理的服务器、1000 行代码、几个小时的时间，可以让你对 GPT 级别的系统有完整的第一手理解。
>
> **可迁移**：对于任何你想真正理解的大型系统，尝试找到它的"124M 参数等价版本"——小到可以完整实现和训练，但保留了所有关键机制。

---

> **案例：Chinchilla 定律改变了整个行业**
>
> **情境**：2022 年，DeepMind 发布了 Chinchilla 论文，结论是大多数当时已发布的大模型（包括 GPT-3、Gopher、Megatron-Turing）都是严重欠训练的
>
> **关键时刻**：他们表明，70B 参数模型（Chinchilla）在 1.4 万亿 token 上训练，性能超过了 280B 参数的 Gopher 在 3000 亿 token 上的结果。更少的参数，但更多的数据，在同等算力下性能更好。
>
> **结果**：整个行业迅速调整策略。之后发布的模型（LLaMA 系列、Mistral 等）都按照 Chinchilla 定律配比参数和数据，性能大幅提升。
>
> **启示**："更大的模型"不等于"更好的模型"——在给定计算预算内，数据量和模型大小的平衡同样重要，甚至更重要。
>
> **可迁移**：当评估或选择语言模型时，不只看参数量，也看训练 token 数。一个参数少但训练充分的模型，可能比参数多但欠训练的模型表现更好。

## 在你的生活中

**把规模定律的洞见转化为实际判断：**

**1. 评估模型时，同时看参数量和训练 token 数**
越来越多的模型提供了这两个数字。Chinchilla 最优配比是 D ≈ 20N。如果一个模型参数很多但训练 token 数很少，它可能是欠训练的；如果训练数据极多但模型很小，它可能已经过了性能平台期。

**2. 理解"更大的模型更聪明"的限制**
规模定律描述的是平均性能，不是特定任务。某些任务（如需要精确事实记忆）可能更受益于更多参数，某些任务（如通用推理）更受益于更多训练数据。在你的具体任务上，小型充分训练的模型有时优于大型欠训练的模型。

**3. 对"涌现能力"保持谨慎乐观**
当前 AI 领域频繁谈论"涌现"——某些能力只有超过特定规模才出现。但研究表明，很多"涌现"现象在使用更细粒度的评估指标时会呈现为平滑的曲线，不是真正的突变。在实际应用中，测试你真正关心的任务，而不是依赖规模推断。

## 与下一章的连接

规模定律描述了给更多计算时发生什么。但还有另一个与"算力"不同的维度：**训练目标的选择**。

当你把 RL 的训练信号从"人类偏好"换成"数学题的正确性"，会发生什么？

DeepSeek R1 的故事告诉我们：会发生令人惊讶的涌现——不是能力的平滑提升，而是全新认知策略的自我发现。

---

**本章框架一句话**：规模定律是现代 LLM 的基本定律——给更多数据和算力，性能可预测地提升；而 Chinchilla 定律告诉我们，平衡比堆砌更重要。
