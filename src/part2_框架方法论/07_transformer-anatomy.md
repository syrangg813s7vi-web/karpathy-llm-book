# 第 7 章：Transformer 的解剖——每个组件存在的理由

> **核心命题**：Transformer 的每个组件都有明确的工程理由。残差连接是梯度高速公路，LayerNorm 稳定训练，前馈网络给 token 独立思考的空间，多头注意力并行捕捉不同关系。没有魔法，全是工程需要。

---

## 思维框架

当你第一次看到完整的 Transformer 架构图，感觉就像看一份复杂的电路图。自注意力、多头、残差连接、层归一化、前馈网络——这些组件是怎么拼在一起的？为什么要这样拼？

Karpathy 的方式是逐个引入每个组件，并在引入时给出它解决的具体问题。

这让 Transformer 从"记忆下来的架构"变成了"理解了设计思路的系统"。你知道每个旋钮控制什么，也知道如果去掉某个组件会发生什么。

核心结构很简洁：一个 Transformer 块 = 注意力子层 + 前馈子层，每个子层都包含层归一化和残差连接。把这个块叠加 N 次，在最后加一个到词汇表大小的线性投影，就是完整的 GPT。

---

## 理论基础

**残差连接：梯度高速公路**

考虑一个 12 层的 Transformer。反向传播时，梯度需要从最后一层传回第一层，经过 12 次矩阵乘法。每次乘法都可能让梯度缩小（消失）或放大（爆炸）。

残差连接的解法优雅而简单：`output = layer(x) + x`。

不是把输入彻底转化成输出，而是**学习一个残差**——在输入基础上需要做什么修正。

这带来了一个关键的梯度特性：在反向传播时，梯度可以通过加法直接流向更早的层，不经过复杂的矩阵乘法。加法操作的梯度是 1，所以不会消减。这条"梯度高速公路"让训练 100 层的网络成为可能。

从初始化角度看也有好处：初始时，如果把每个子层的输出初始化为接近 0（通过缩小输出投影的初始权重），整个网络行为接近恒等函数，训练更稳定。

**层归一化（Layer Normalization）**

与 BatchNorm 对一批样本的同一特征做归一化不同，LayerNorm 对**单个样本的所有特征**做归一化。

公式：`y = (x - mean(x)) / std(x) * γ + β`，其中 γ 和 β 是可学习参数。

为什么 Transformer 用 LayerNorm 而不是 BatchNorm？

BatchNorm 在小批量或序列长度可变时效果差，而且训练和推理行为不同（训练时用批次统计，推理时用运行统计）。LayerNorm 完全在单个样本上操作，没有批次耦合，训练推理行为一致。

**现代 Transformer 用 Pre-Norm（前置归一化）**：归一化放在注意力/FFN 之前而不是之后。实验表明 Pre-Norm 训练更稳定，梯度流动更好。

**前馈网络（FFN）**

每个 Transformer 块里，注意力之后还有一个两层 MLP：

```
FFN(x) = Linear(GELU(Linear(x, 4d)), d)
```

第一层把维度从 d 扩展到 4d，第二层再压缩回 d。

为什么需要 FFN？注意力机制处理的是 token 之间的**关系**（谁应该关注谁），而 FFN 处理的是每个 token 的**独立变换**（这个表示应该变成什么）。两者互补：注意力是聚合，FFN 是提炼。

4x 扩展因子是经验常数，在 GPT-2 到 GPT-4 的所有版本中基本保持不变。

**Token 嵌入 + 位置嵌入**

输入的每个 token 有两个向量表示：
- **Token 嵌入**：词汇表中每个 token 对应一个可学习的 d 维向量，查找表操作。
- **位置嵌入**：每个位置（0, 1, 2, ..., T-1）也对应一个可学习的 d 维向量。

两者相加，输入到第一个 Transformer 块。这样模型既知道"这个词是什么"，也知道"这个词在序列的哪个位置"。

## 案例

> **案例：从 10M 参数的玩具模型到 GPT-2 的连续性**
>
> **情境**：Karpathy 在"Let's Build GPT"教程结尾，把刚构建的 ~10M 参数字符级模型和真实的 GPT-2 放在一起对比
>
> **关键时刻**：他展示了 GPT-2 124M 的架构超参数：12 层、12 头、768 维嵌入。然后把这些数字代入他们一步步构建的代码里——除了 BPE 分词替换字符级分词、训练数据从 Tiny Shakespeare 换成真实互联网文本——其他完全一样。
>
> **结果**：学生们意识到：ChatGPT 不是另一个物种，它只是同一个架构在更大数据和更多算力上的运行结果。这个认知消除了大量不必要的敬畏感。
>
> **启示**：理解了 10M 参数的教学模型，就理解了 GPT-4 的核心机制——规模不同，原理相同。
>
> **可迁移**：在工程工作中，当你面对看似复杂的系统，先找它最简单的版本，理解通了再扩展规模。

---

> **案例：残差连接对深层网络训练的影响**
>
> **情境**：早期的深度学习（ResNet 之前），训练超过 20 层的网络极其困难——增加层数反而会让性能下降（不是过拟合，是训练集性能也下降）
>
> **关键时刻**：ResNet 论文（He et al., 2016）引入残差连接，首次让训练 100+ 层的网络成为可能。Karpathy 在构建 GPT 时沿用了这个设计，并在代码中明确展示了没有残差连接时梯度如何消失。
>
> **结果**：带残差连接的 6 层 Transformer 能够正常收敛，等效的无残差版本在训练几千步后梯度范数就变得很小，深层网络实际上没有在学习。
>
> **启示**：残差连接不是优化技巧，是训练深层网络的必要条件。
>
> **可迁移** [通识补充]：残差连接的思想已经扩展到所有深度学习领域。在设计自定义神经网络时，如果网络超过 4-5 层，加上残差连接几乎总是有益的。

## 在你的生活中

**建立对 Transformer 架构的完整直觉：**

**1. 数出 GPT-2 的参数**
GPT-2 124M：12 层、12 头、嵌入维度 768、词汇表 50257、最大序列长度 1024。手算参数量：词汇嵌入 50257×768 = 38.6M，位置嵌入 1024×768 = 786K，每个 Transformer 块约 7.1M×12 = 85.2M。总计约 124M。当你能从超参数推算出参数量，你对架构的理解就已经进入工程级别了。

**2. 理解 Pre-Norm vs Post-Norm 的实际影响**
查一下 GPT-2 原始论文和现代实现（比如 llama.c）的区别：原始 GPT-2 用 Post-Norm，后来的 GPT-3 和几乎所有现代模型改为 Pre-Norm。尝试在你的实现里切换，观察训练稳定性的差异。

**3. 用 PyTorch 写一个 Transformer 块（1-2 小时）**
按照这章描述的结构：`LayerNorm → MultiHeadAttention → Residual + LayerNorm → FFN → Residual`，用 PyTorch 模块组合实现。验证前向传播的输出形状正确（B, T, C → B, T, C）。不需要训练，只需要跑通前向传播。

## 与下一章的连接

我们理解了单个 Transformer 模型的内部结构。但一个模型是怎么从"原始参数"变成"ChatGPT"的？

这中间有三个截然不同的训练阶段，每个阶段都在做完全不同的事情。理解这三个阶段，才能理解为什么基础模型和助手模型行为差异如此之大。

---

**本章框架一句话**：Transformer 的每个组件都有明确的工程理由——残差连接保证梯度流动，LayerNorm 稳定训练，FFN 给每个 token 独立提炼的空间。
