# 第 1 章：神经网络不是黑魔法，是可微分程序

> **核心命题**：神经网络是由可微分操作组成的程序，梯度下降是优化它的通用方法。理解这一点，AI 就不再神秘——它变成了可以从零构建的工程系统。

---

## 思维框架

在 Karpathy 的所有讲座中，有一个立场贯穿始终：他拒绝把神经网络当作"黑盒"或"魔法"来对待。

他的切入点总是一样的：**从零开始写代码**。不是调用 API，不是加载预训练模型，而是从最简单的基础组件——矩阵乘法、softmax、负对数似然损失——一行一行地搭建起整个系统。

这种方式背后有一个深刻的信念：只有真正构建过某样东西，才能真正理解它。

神经网络从这个视角看，是什么？

**它是一个可微分程序。** 你定义一组操作（矩阵乘法、激活函数、规范化），这些操作串联在一起将输入转化为输出。每个操作都有导数。整个程序的输出相对于每个参数的梯度，可以通过链式法则自动计算出来。

然后你做一件简单的事：沿着梯度的反方向，稍微移动参数——这叫梯度下降。重复这个过程成千上万次，程序就"学会"了产生你期望的输出。

这不是魔法。这是微积分加上大量算力。

---

## 理论基础

**为什么这个框架是对的？**

因为神经网络的每个组成部分，都可以追溯到一个明确的数学操作：

**前向传播**是函数组合：输入 x 经过一系列可微函数 f₁, f₂, ..., fₙ 的组合，产生输出 y。

**损失函数**是测量输出与期望之间距离的标量。对于语言模型，这通常是**负对数似然**（Negative Log-Likelihood，NLL）：给定上下文，模型预测下一个 token 的概率分布，损失就是正确 token 概率的负对数。

**反向传播**是链式法则的递归应用。你从损失出发，一路向后计算每个参数对损失的偏导数。每个操作都有一个正向实现和一个反向（计算梯度的）实现。

**梯度下降**用梯度信息更新参数：θ ← θ - α∇L，其中 α 是学习率。

这个框架的力量在于它的普适性。只要你能写出一个前向函数，并且它是处处可微的（或几乎处处可微的），梯度下降就能优化它。这就是为什么同一个框架——PyTorch——可以用来训练图像分类器、语言模型、蛋白质结构预测器，甚至游戏智能体。

## 案例

> **案例：从零构建反向传播引擎——micrograd**
>
> **情境**：Karpathy 想让学生真正理解反向传播是什么，而不只是"调用 `.backward()`"
>
> **关键时刻**：他决定从最基础的层面开始——写一个仅支持标量运算的自动微分引擎，代码只有约 100 行 Python。每个数字被包装成一个 `Value` 对象，每次运算都记录"我是怎么来的"（来自哪两个父节点、经过什么操作）。调用 `.backward()` 时，从输出向输入递归地应用链式法则。
>
> **结果**：这个微型引擎（他命名为 micrograd）能够训练一个小型神经网络——而整个实现清晰到任何人都能一行一行读懂。
>
> **启示**：PyTorch 的 `.backward()` 做的事情，和这 100 行代码做的事情，在原理上完全相同。规模不同，原理相同。
>
> **可迁移**：任何时候当你感觉某个系统"太神秘"，考虑用最小可行版本重建它。复杂性往往隐藏了简单的核心原理。

## 在你的生活中

**明天可以做的三件事：**

**1. 手算一次反向传播**
取一个简单的网络：一个线性层 y = wx + b，损失 L = (y - target)²。手算 ∂L/∂w 和 ∂L/∂b。这个计算只需要几分钟，但它会让你对所有框架中 `.backward()` 的行为有直觉性理解。

**2. 用 micrograd 构建一个神经元**
Karpathy 的 micrograd 代码公开在 GitHub 上（约 100 行）。把它克隆下来，跑通例子，然后在里面加一个新操作（比如 ReLU），并验证梯度的数值正确性（用有限差分法检验）。

**3. 换个方式看待 AI 工具**
下次使用 ChatGPT 或 Claude 时，不要把它当成神谕来问。把它当成一个被训练来完成 token 预测任务的程序，想想：这个问题，对于一个"在互联网上见过大量类似文本"的预测引擎来说，是容易回答的还是困难的？
