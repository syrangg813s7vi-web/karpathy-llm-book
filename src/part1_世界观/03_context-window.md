# 第 3 章：上下文窗口是唯一的现实

> **核心命题**：LLM 是无状态的 token 处理器，上下文窗口是它能看到的全部现实。学会操控上下文——知道放什么、怎么放、放多少——是使用 LLM 最核心的技能。

---

## 思维框架

有一个细节，大多数人在使用 ChatGPT 时从未意识到：**每次对话结束后，模型对你没有任何记忆**。

不是"记忆不好"，而是根本没有记忆机制。Karpathy 在讲座里有一句话说得非常直白：

> "这个模型没有持续存在的方式。它启动，处理 token，关闭。对每一个人都是这样。它只是建立一个上下文窗口里的对话，然后一切都被清除了。"

理解这一点，很多"为什么 LLM 会这样"的困惑就消失了。

模型不是一个等着你来找它聊天的朋友。它更像一台无状态的自动售货机：你投入硬币（token），它吐出结果，然后重置，对下一个客户它完全不记得你是谁。

**上下文窗口**（Context Window）是这台机器唯一能看到的东西。它是一个固定长度的 token 序列——现代模型通常支持 4k 到 200k token——包含你的指令、你们的对话历史、你粘贴的文档，所有的一切。

这个窗口之外的，不存在。

---

## 理论基础

**上下文窗口的结构**

在技术层面，一次对话的上下文窗口通常包含三类内容：

**System Prompt（系统提示）**：在对话开始前就被填入的配置信息。这是一块"特权"区域——告诉模型它是谁、该怎么表现、有什么限制。你在 ChatGPT 里设置的"自定义说明"、企业部署时配置的行为规范，都通过这里传递。

**用户消息与助手消息**：对话的实际内容，交替排列。模型看到的不是一问一答，而是一个完整的 token 序列，其中有特殊标记区分谁在说话。

**工具调用结果**：如果模型调用了搜索引擎、代码解释器或其他工具，工具的返回结果也会被注入回上下文，模型看到它们并继续生成。

**上下文窗口 = RAM（内存）的类比**

Karpathy 提出了一个特别有助于理解的类比：把 LLM 想象成一种新型操作系统。

在这个类比里，**上下文窗口就是内存（RAM）**。模型的权重（140GB 的参数文件）是硬盘——有大量信息，但访问是间接的、有损的。上下文窗口才是处理器当前实际在读写的工作区。

任何你希望模型能"看到"的信息，都必须在这个工作区里。这不是建议，这是物理约束。

**上下文窗口的长度限制**

为什么不把窗口做到无限大？因为 Transformer 的自注意力计算复杂度是 O(n²) 的——序列长度翻倍，计算量翻四倍。200k token 的窗口已经是工程上的巨大挑战，1M 的窗口在计算上极其昂贵。

这个约束是真实的，它意味着你需要**主动管理上下文**：什么应该留在对话里，什么可以截断，什么应该通过向量检索（RAG）按需载入。

## 案例

> **案例：直接粘贴 vs 依赖记忆**
>
> **情境**：Karpathy 使用 Claude 分析一篇关于基因组学大型语言模型（Evo 2）的最新学术论文。这篇论文发布于 2025 年，显然不在模型的训练数据里。
>
> **关键时刻**：他没有问"你知道 Evo 2 模型吗？"——那样得到的会是幻觉。他直接把 30MB 的 PDF 文件拖进了 Claude 的对话框，然后问具体的技术问题。
>
> **结果**：模型基于上下文窗口里的真实内容，给出了高质量的技术解读——精确引用了论文中的数据和方法，而不是编造一个"看起来合理"的答案。
>
> **启示**：绕过"有损压缩"的最有效方法，是把信息直接放进上下文窗口，让模型"读"而不是"回忆"。
>
> **可迁移**：处理任何专业文档时——合同、报告、论文、代码库——优先上传或粘贴原文，而不是期望模型知道它。

---

> **案例：系统提示的隐藏配置**
>
> **情境**：企业部署 LLM 客服时，需要让模型始终保持特定的品牌语气、不讨论竞争对手、只回答产品相关问题。
>
> **关键时刻**：工程师在系统提示里写入：角色设定、禁止话题、回答格式要求，这些内容用户看不到，但模型在生成每一个回复时都在遵守。
>
> **结果**：同一个基础模型在不同系统提示下，行为可以完全不同——从旅行助手到代码助手到心理咨询助手。
>
> **启示**：系统提示是调控 LLM 行为的最强杠杆。掌握它，你能将同一个模型定制为完全不同的应用。
>
> **可迁移**：在个人工作流中，为常见任务建立"固定系统提示"——你的角色、输出格式、语言风格要求——每次开始工作时自动加载，省去反复解释的时间。

## 在你的生活中

**立刻可以改变的三个上下文习惯：**

**1. 粘贴原文，而不是描述原文**
下次你需要 LLM 分析一份文档，不要说"我有一份关于 XX 的报告"然后再描述它。直接把文档内容粘贴进去。这个习惯能让回答质量提升一个数量级。

**2. 为长对话建立"上下文刷新"机制**
当你和模型对话很长时间后，上下文窗口可能已经充满了旧的讨论，模型开始"忘记"早期的重要约束。养成定期整理的习惯：在新对话开始时，把最重要的背景信息和约束重新陈述一次。

**3. 把少样本示例放在提示的开头**
当你需要特定格式的输出时，给模型看 2-3 个例子比描述格式要求更有效。把这些示例放在系统提示或对话最开头——它们是上下文窗口里权重最高的部分（因为所有后续内容都要从这个基础上生成）。

我们已经建立了三个核心认知：神经网络是可微分程序，LLM 的知识是有损压缩，上下文窗口是模型的唯一现实。
